# Level 2: 주요 기능별 상세 흐름도 (Feature Details)

이 문서는 뉴스 큐레이션 플랫폼의 핵심 기능인 "뉴스 수동 추가"와 "RSS 피드 조회"의 상세한 프로세스를 나타냅니다.

## 2.1 뉴스 수동 추가 흐름 (Add News)

사용자가 URL을 입력하면 서버가 해당 URL을 스크래핑하고 AI를 통해 요약한 뒤 DB에 저장하는 과정입니다.

```mermaid
flowchart TD
    A1((사용자)) -->|URL 입력 및 추가 버튼 클릭| B1[프론트엔드 UI]
    B1 -->|POST /api/add-news| C1[백엔드 서버]
    
    C1 -->|HTTP GET 요청| D1[뉴스 웹사이트]
    D1 -->|HTML 문서 반환| C1
    
    C1 -->|Cheerio 파싱 및 본문 추출| E1[백엔드 서버 내부 로직]
    E1 -->|본문 텍스트 프롬프트 전달| F1[AI 요약 엔진]
    F1 -->|4줄 요약 및 번역 반환| C1
    
    C1 -->|제목, 이미지, 요약문 저장| G1[(데이터베이스)]
    G1 -->|저장 성공| C1
    
    C1 -->|신규 뉴스 데이터 응답| B1
    B1 -->|화면 리스트 갱신| A1
```

## 2.2 RSS 뉴스 검색 흐름 (Discover RSS)

시스템에 등록된 언론사들의 최신 RSS 피드를 실시간으로 불러와 사용자에게 보여주는 과정입니다.

```mermaid
flowchart TD
    A2((사용자)) -->|RSS 피드 채널 선택| B2[프론트엔드 UI]
    B2 -->|GET /api/rss?url=...| C2[백엔드 서버]
    
    C2 -->|XML 데이터 요청| D2[언론사 RSS 서버]
    D2 -->|XML 데이터 반환| C2
    
    C2 -->|rss-parser 라이브러리로 분석| E2[데이터 정제 로직]
    E2 -->|불필요한 태그 및 이미지 필터링| E2
    
    E2 -->|정제된 JSON 데이터 구조 반환| B2
    B2 -->|디스커버리 탭에 리스트 렌더링| A2
```
